# Relativa CRM — Монетизаційні рівні ендпоінтів

> Цей документ класифікує всі 101 API-ендпоінт за трьома ціновими рівнями.
> Використовуйте його як основу для реалізації rate-limiting middleware та feature-gates в API Gateway.

---

## Позначення рівнів

| Символ | Рівень | Опис |
|--------|--------|------|
| 🆓 | **Free** | Повністю доступно без оплати — без обмежень за обсягом |
| 💎 | **Pro** | Потребує підписки Pro для доступу |
| ♾️ | **Unlimited** | Доступно на всіх рівнях, але з rate-limiting на Free/Pro; план Unlimited знімає всі обмеження |

> Ендпоінти позначені ♾️ завжди доступні, але повертають `429 Too Many Requests` при перевищенні квоти рівня.

## 3. Організації — 🆓 Free / ♾️ Unlimited

**Базовий шлях:** `/api/v1/organizations`

**Обґрунтування:** Створення та керування організацією — перша дія нового користувача. Базові CRUD-операції безкоштовні, щоб усунути тертя під час онбордингу. Масове створення користувачів (`POST /{orgId}/users`) обмежено за кількістю, оскільки спричиняє навантаження на БД та доставку email.

| Рівень | Метод | Шлях | Примітка |
|--------|-------|------|----------|
| 🆓 | `POST` | `/api/v1/organizations/` | Створення організації |
| 🆓 | `GET` | `/api/v1/organizations/` | Список організацій користувача |
| 🆓 | `GET` | `/api/v1/organizations/search` | Пошук за назвою |
| 🆓 | `GET` | `/api/v1/organizations/{id}` | Деталі організації |
| 🆓 | `PUT` | `/api/v1/organizations/{id}` | Оновлення організації |
| 🆓 | `GET` | `/api/v1/organizations/{id}/settings` | Читання налаштувань |
| 🆓 | `PUT` | `/api/v1/organizations/{id}/settings` | Оновлення налаштувань |
| ♾️ | `POST` | `/api/v1/organizations/{organizationId}/users` | Масовий імпорт користувачів — Free: 50/день · Pro: 500/день · Unlimited: без обмежень |
| 🆓 | `PATCH` | `/api/v1/organizations/{organizationId}/users/{userId}` | Оновлення профілю користувача |
| 🆓 | `DELETE` | `/api/v1/organizations/{organizationId}/users/{userId}` | Видалення користувача |
