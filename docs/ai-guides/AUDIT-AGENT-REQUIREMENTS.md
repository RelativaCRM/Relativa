# Audit Agent Requirements

> **Last verified:** 2026-05-02

## Mandatory Rule For Any AI Agent

Any AI agent working in this repository must enforce audit requirements for persistence changes:

1. Before changing DB entities/tables, check [AUDIT-COVERAGE-MATRIX.md](AUDIT-COVERAGE-MATRIX.md).
2. If an existing touched table requires audit, preserve or extend audit writes.
3. If a new entity/table is added, the same task must also add:
   - matching audit table/configuration, and
   - audit write event flow (fire-and-forget to Audit service).
4. A task that introduces a new table/entity without audit is incomplete.
5. Update relevant guides listed in [AI-GUIDES-INDEX.md](../../AI-GUIDES-INDEX.md), including this file's date if behavior changes.

## Enforced Checklist

- `requires_audit` decision recorded in coverage matrix
- outbox event generated by writer service
- Audit consumer persists event to audit table
- migration includes schema changes
- docs updated
